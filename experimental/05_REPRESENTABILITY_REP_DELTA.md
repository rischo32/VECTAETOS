# Representability Predicate Rep(Δ)

Status: Critical open specification  
Purpose: Make the softest part of the formal system explicit.

---

## 1. Current Definition

```text
Rep(Δ) ∈ {0,1}
```

where:

```text
Rep(Δ)=1
```

iff Δ can exist as a representable epistemic curvature configuration.

---

## 2. What Rep Is Not

Rep is not:

- metric
- score
- classifier
- runtime authority
- optimizer
- preference
- deployment threshold
- reward function
- policy

---

## 3. Current Failure Modes

Rep(Δ)=0 if Δ:

- privileges one singularity as dominant authority
- privileges one triality axis
- collapses 4ES plurality
- induces prescriptive closure
- destroys global relational readability
- cannot sustain carrier status
- makes projection impossible without false authority
- converts uncertainty into forced certainty

---

## 4. Minimal Formal Strategy

Rep can remain a primitive predicate if its role is clearly declared.

Recommended phrasing:

```text
Rep is a primitive ontological predicate in VECTAETOS 1.x.
It defines the non-algorithmic representability boundary.
Audit tools may approximate Rep, but no approximation becomes Rep.
```

This is acceptable if the project is explicit that Rep is not yet constructively complete.

---

## 5. Constructive Approximation Strategy

Build approximate detectors:

### A1. Dominant singularity detector

Input:

```text
Δ ∈ C²
```

Detect whether curvature mass concentrates around one singularity.

Possible audit quantity:

```text
D_i = Σ_{j<k, i∈{i,j,k}} |Δ_{ijk}|
```

Risk:

This is a metric, therefore cannot define Rep canonically.

Use only as audit approximation.

---

### A2. Triality imbalance detector

Compute:

```text
||Δ - P𝒯Δ||
```

If nonzero, triality balance fails.

This is closer to formal structure.

---

### A3. Closure pressure detector

Detect whether Δ collapses multiple admissible projections into one forced output.

This is currently conceptual, not formal.

---

### A4. Connectivity/readability detector

Build graph from significant curvature components and test fragmentation.

This approximates global relational readability.

---

## 6. Canonical vs Approximate Separation

Canonical:

```text
Rep(Δ)
```

Audit approximation:

```text
Rep_hat(Δ)
```

Rule:

```text
Rep_hat does not define Rep.
Rep_hat only produces evidence.
```

---

## 7. Suggested Formal Layer

Define:

```text
Rep(Δ)=1
```

iff all canonical conditions hold:

```text
C1: Δ ∈ Im(d₁)
C2: d₂Δ=0
C3: P𝒯Δ=Δ
C4: no ontological dominance condition is violated
C5: carrier readability is preserved
```

But C4 and C5 still need formalization.

---

## 8. Red-Team Cases

Rep must fail for:

1. Singular dominance:
   - one Σ controls most curvature.
2. Triality collapse:
   - Δ valid algebraically but not triality-balanced.
3. Prescriptive closure:
   - all projections collapse into one forced directive.
4. Destructive coherence:
   - configuration is internally coherent but destroys carrier conditions.
5. Audit authority injection:
   - Rep_hat treated as Rep.
