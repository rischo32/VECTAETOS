SECURITY

Reporting a Vulnerability

If you believe you have found a security issue related to the infrastructure or code of this repository, please report it responsibly.

Contact:

- Email: <your-email-here>

Please include:

- description of the issue
- steps to reproduce
- potential impact

---

Scope

This policy applies to:

- repository code
- CI/CD workflows
- scripts and infrastructure components

---

Out of Scope

The following are not security vulnerabilities:

- epistemic outputs
- runic projections
- lack of answers or QE states
- non-prescriptive behavior
- absence of optimization or decision-making

These are intentional properties of the system.

---

Important Distinction

Vectaetos defines security at two levels:

- Infrastructure security → handled here
- Ontological security → defined in "/formal/SECURITY_MODEL.md"

The latter is not subject to vulnerability reporting, as it does not operate as an enforceable mechanism.

---

Responsible Disclosure

Please do not open public issues for security vulnerabilities.

We aim to respond within a reasonable timeframe.
