## Notes

- **Status labels**: “Active” means the guard is implemented in the current `GUARD_TABLE.md`【861243860326949†screenshot】. “Planned” means described in the final hardened model or workplan but not yet implemented.
- **Evidence classes** are taken from the unified finding schema: E0 (text claim), E1 (static scan), E2 (AST/contract compliance), E3 (deterministic test suite), E4 (empirical validation), E5 (external replication)【861243860326949†screenshot】.
- **Enforcement modes**: `Strict` triggers CI failure on drift; `Report` logs findings without blocking. Many guards support both modes depending on configuration.
- **Integrity posture** reflects whether the guard operates on immutable anchors, runtime state, semantic vocabulary or evidence posture as introduced in the upgraded model.  It highlights that byte integrity is distinct from semantic truth【289806268914716†screenshot】.

This table provides a consolidated map of the perimeter guard ecosystem for VECTAETOS.  It should be updated when new guards are implemented or planned guards change status.  The table itself is descriptive and non‑authoritative; it does not define the ontology or safety guarantees.
