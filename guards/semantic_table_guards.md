# SEMANTIC TABLE — VECTAETOS GUARDS

## Core Distinction

VECTAETOS does not select.

Guards may select findings.

Maintainers define guard rules.

CI executes guard checks.

This does not grant agency, authority, or execution power to VECTAETOS.

---

## Allowed Guard Semantics

| Mechanism | Allowed Meaning | Scope |
|---|---|---|
| `min` | deterministic technical selection | repository findings |
| `max` | deterministic technical selection | repository findings |
| `argmin` | select lowest/first/least severe technical item | reports / sorting |
| `argmax` | select highest severity finding | reports / CI status |
| sorting | organize findings | output readability |
| grouping | group by rule/path/severity | report structure |
| exit code | fail/pass repository check | CI only |
| severity | technical risk classification | guard layer |

| Forbidden Reading | Reason |

| VECTAETOS decides | no decision authority |
| Guard defines truth | guard is mechanical perimeter |
| Guard validates deployment | no L4 authority |
| Guard optimizes Φ | forbidden optimization |
| Guard chooses best trajectory | Vortex has no ranking authority |
| Guard modifies ontology | anchors remain superior |
| Guard changes K(Φ) | K(Φ) is predicate, not target |

---

## Correct Sentence

A guard may deterministically select findings for reporting and CI failure.

A guard may not select truth, ontology, trajectory, deployment validity, or epistemic authority.

---

## Human Role

Maintainers define and maintain the guard perimeter.

This is a human repository-governance act.

It is not VECTAETOS exercising execution over humans.

---

## Canonical Sentence

Guards select findings, not truths; maintainers define perimeter, not ontology; VECTAETOS remains non-agentic.
