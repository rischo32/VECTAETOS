## Purpose

The `knowledge_base/` directory stores auxiliary reference material used for orientation, terminology, historical context, research comparison, and semantic mapping.

It may contain:

- philosophical dictionaries,
- external terminology notes,
- historical PDFs,
- research references,
- non-canonical summaries,
- extracted terminology,
- comparative notes,
- source maps,
- reading indexes.

Its role is supportive.

It is not authoritative.

---

## Canonical Boundary

VECTAETOS terminology is defined only by the canonical anchor set, frozen formal documents, the master index, and explicit project decisions.

The knowledge base may help explain terms, but it may not define them.

Correct use:   knowledge_base = auxiliary reference layer

Repository Perimeter Rule
The current repository phase is:

fundamental repository perimeter alignment
During this phase, the knowledge base is used only for mapping and recalibration.
It must not introduce new ontology.
It must not create new operational layers.
It must not influence Vortex, ASIMULATOR, or ASI_MOD expansion.
Guard Compatibility
Guard scripts may scan or reference the knowledge base, but findings inside knowledge_base/ should be classified carefully as:

external reference
historical wording
non-canonical terminology
possible errata candidate
active drift only if reused outside knowledge_base
A drift inside the knowledge base is not automatically a repository violation.
A drift copied from the knowledge base into active documentation may become a violation.
