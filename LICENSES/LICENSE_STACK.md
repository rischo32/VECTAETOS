# VECTAETOS™ License Stack

**Status:** Draft  
**Version:** 0.1  
**Role:** Map of license layers and applicability  
**Legal note:** Not legal advice.

## 1. Purpose

The VECTAETOS™ License Stack separates copyright, implementation behavior, projection use, and trademark / compatibility claims.

This prevents a single license from collapsing all layers into one authority.

```text
ontology layer       → VCL-2.0
software layer       → AEPL-2.0-VECTAETOS
implementation layer → VNAL-1.1
projection layer     → VPL-1.0
identity layer       → VTP-1.0
```

## 2. Supersession Map

```text
AEPL-1.2-VECTAETOS  → historical ethical-use layer
VCL-1.0             → historical canonical custom layer
VNAL-1.0            → implementation behavioral layer
VCL-2.0             → proposed canonical ontology license
AEPL-2.0            → proposed software / tooling license
VNAL-1.1            → proposed non-agentic implementation license
VPL-1.0             → proposed projection / trace license
VTP-1.0             → proposed identity / compatibility policy
```

This map does not retroactively change external DOI records unless a new release explicitly declares supersession.

## 3. Core Sentence

The license stack protects repository use, derivative claims, and public identity.

It does not define Φ.

It does not prove truth.

It does not authorize deployment.

It does not make any artifact an authority.
