# VECTAETOS™ License Lineage and Supersession

**Status:** Draft  
**Version:** 0.1  
**Role:** Historical and operational map of VECTAETOS license evolution  
**Legal note:** Not legal advice.

## 1. Purpose

This document explains how historical VECTAETOS™ licenses and the proposed license stack relate to one another.

It does not retroactively modify external DOI records, archived releases, papers, PDFs, or prior license grants.

## 2. Historical Layers

### AEPL-1.2-VECTAETOS

Historical ethical-use layer.

Primary function:

```text
restrict harmful, coercive, military, punitive, deceptive, or manipulative use
preserve epistemic and ethical processing boundaries
```

Status in new stack:

```text
historical root
superseded for new releases by AEPL-2.0-VECTAETOS where explicitly declared
```

### VCL-1.0

Historical custom VECTAETOS license layer.

Primary function:

```text
protect canonical identity
preserve non-authoritative ontology framing
separate canonical core from derivative claims
```

Status in new stack:

```text
superseded for new canonical releases by VCL-2.0 where explicitly declared
```

### VNAL-1.0

Historical non-authoritative / implementation behavioral layer.

Primary function:

```text
prevent single-answer collapse
prevent ranking and reward-loop integration
preserve uncertainty and QE
```

Status in new stack:

```text
superseded for new implementation releases by VNAL-1.1 where explicitly declared
```

## 4. Supersession Rule

A new license supersedes an older license only when a file, release, DOI record, directory, or publication explicitly states that supersession.

Recommended statement:

```text
This release is governed by VCL-2.0 and supersedes VCL-1.0 for this release only.
Prior DOI records remain governed by the license declared in those records unless otherwise stated.
```

## 5. Non-Retroactivity

The new stack does not automatically rewrite Zenodo records, published PDFs, archived DOI releases, third-party copies, previous repository snapshots, or prior license grants.

## 6. Canonical Separation

License lineage is not ontology lineage.

```text
license version ≠ ontology version
legal supersession ≠ semantic supersession
DOI update ≠ Φ mutation
```
